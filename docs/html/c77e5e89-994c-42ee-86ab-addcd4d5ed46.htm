Message Filtering

Use events provided by RemoteAgencyManagerTNetworkMessage, TSerialized, TEntityBase to filter messages.

Events

RemoteAgencyManagerTNetworkMessage, TSerialized, TEntityBase includes 2 events:

  • AfterMessageReceived

    This event will be raised after manager unpack the message, before pass it to proxy or service wrapper.

  • BeforeMessageSending

    This event will be raised after manager received a message from proxy or service wrapper, before pack and send it out.

Event Argument

An object of BeforeMessageProcessingEventArgsBaseTSerialized will be passed to handler as argument of event.

This object contains full information from MessageInstanceMetadata, as well as serialized message contains parameters and/or return value of asset.

  • When IsException is set to true, this object is an instance of BeforeExceptionMessageProcessingEventArgsTSerialized. Type of original exception is specified.

  • When IsException is set to false, this object is an instance of BeforeMessageProcessingEventArgsTSerialized. Used generic types are specified when asset is generic supported.

BeforeMessageProcessingEventArgsBaseTSerialized is inherited from BeforeMessageProcessingEventArgsBase.

In event handler method, FurtherProcessing should be specified. Value is from MessageFurtherProcessing, default is Continue.

Further Processing

Further processing after event handler is based on the value of FurtherProcessing.

  • Continue

    Continues to process this message. This is the default value.

  • ReplacedWithException

    Replaces the message as an exception instance of MessageProcessTerminatedException and continues.

    This should be used only for response message, like the returning of a service method, or the receiver proxy or service wrapper will drop this message.

  • TerminateSilently

    Terminates this message instance silently.

    This should be used only for one way request message, or the requester may be locked in response message waiting when MillisecondsTimeout is absent.

  • TerminateWithExceptionReturned

    Terminates this message and send an exception instance of MessageProcessTerminatedException back to the sender.

    This should be used only for request message with no one way set. Trying set FurtherProcessing to this value with one way present, will set it as TerminateSilently instead.

Message Waiting

Proxy or service wrapper will lock and wait, until or unless:

  • Linked response message is received.

  • Linked exception message is received.

  • This asset is set as one way by using CustomizedOneWayOperatingAttribute and it is applicable.

    Note: for example, one way attribute will be ignored when set to a method with returning type or out parameter.

  • Timeout is specified by CustomizedOperatingTimedoutTimeAttribute and it is passed.

Or, it will wait forever until the thread of the caller ends.
