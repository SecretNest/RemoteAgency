Managing Objects (Proxy and Service Wrapper)

Adding and removing proxy and service wrapper objects.

General

You can choose to manage the lifespan of the proxy and service wrapper objects your own, or by manager instance.

  • If you want to manage the lifespan your own:

    • Use ProxyCreator or ServiceWrapperCreator to create the object.

    • Add them to manager.

    • You can choose whether to dispose the object when removal from manager. If you choose not to, you need to dispose them by your own code when necessary.

    It's possible to add an object to multiple manager instances, but it's not recommended.

  • If you want the manager instance to manage the lifespan:

    • Calling add methods with the object of ProxyCreator or ServiceWrapperCreator, will create the object and add it to manager in one process.

    • Manager will handle the disposing process when object removing.

    This way is easier and recommended.

Add and query proxy objects

To create a proxy object manually, you can use CreateProxyObjectTInterfaceContract.

To add a proxy object to manager, you can call one of these methods:

  • AddProxyTInterfaceContract

  • AddProxyTInterfaceContract

  • AddProxyTInterfaceContract

When instance id absent while creating proxy object, a randomized value will be used.

Use GetAllProxies and GetAllProxiesTInterfaceContract to query the proxies managed by this manager instance.

Add and query service wrapper objects

To create a service wrapper object manually, you can use CreateServiceWrapperObjectTServiceObject or CreateServiceWrapperObjectTServiceObject, based on whether one or multiple service contract interfaces are implemented by this service object.

To add a service wrapper object based on one service contract interface to manager, you can call one of these methods:

  • AddServiceWrapperTInterfaceContract

  • AddServiceWrapperTInterfaceContract

  • AddServiceWrapperTInterfaceContract

To add a service wrapper object based on multiple service contract interfaces to manager, you can call one of these methods:

  • AddServiceWrapper

  • AddServiceWrapperTServiceObject

  • AddServiceWrapperTServiceObject

When multiple service contract interfaces are implemented by one service object, it's recommended to create one service wrapper to cover all interfaces, especially when you use caching function.

Use GetAllServiceWrappers and GetAllServiceWrappersTInterfaceContract to query the service wrappers managed by this manager instance.

Remove

  • To remove one object from manager, use RemoveManagingObject.

  • To remove all objects from manager, use RemoveAllManagingObjects.

  • To remove one service contract interface from a service wrapper object, use RemoveTypeFromServiceWrapper.
