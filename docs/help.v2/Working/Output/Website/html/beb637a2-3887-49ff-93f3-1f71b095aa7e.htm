Name Customizing

Due to the requirement of serialization of messages and type building, names need to be chosen for each type, asset and parameters. By default, those names are generated automatically based on original names.

Attributes

Developer could change those names by specifying the replaced name using attributes. Check the list of attributes for details.

Collision Avoidance

When more than one asset or parameter intend to use the same name, method overloading for example, auto collision avoidance is taken place.

It should be noted that the name given clearly by attribute will not be modified due to collision avoidance. When an auto chosen name is the same as the manually given one, the auto chosen name will be changed. If collision is found among names specified by attributes, exception will be thrown while creating types.
