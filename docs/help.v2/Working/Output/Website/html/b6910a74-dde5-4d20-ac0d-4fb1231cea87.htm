Remote Agency

Async Method

Asynchronous method is not supported by Remote Agency because the advantage from asynchronous calling is meaningless compared to the overhead when running code remotely.

AsyncMethodAttribute can be used on all methods with return as Task, ValueTask or their derived class for compatible when calling asynchronous methods.

Simulation

Through supporting of asynchronous method is insignificant in Remote Agency. Simulation solution presents still for code level compatible reason.

Due to lack of serialization support of Task, the method returning an instance of Task could not be used in Remote Agency directly. However, the method marked with AsyncMethodAttribute is supported by Remote Agency by extracting the result on remote site and reconstruct the task instance from it on local.

 Note
By getting the result of the Task on the remote side, the exception thrown inside the task will be processed and transmitted to local. Exception will be thrown on local when getting the return value from the method if happened, instead of getting Result property or running Wait method of the Task instance.
Mechanism

No matter async keyword is used on the method or not, this attribute marked method will be treated in the following ways:

  • Normally, the server side (service wrapper) will call this method and wait for the completion, when requested from the client side (proxy).

  • If result is required, a completed task with the result from the server side will be created on the client side.

  • The state of the task will never be transferred.

  • If there is any exception thrown inside the method, the exception object will be transferred to the client side and thrown. No exception handling is called on the server side.

  • If the method is marked with one-way, server side will call the method without waiting for the result.
