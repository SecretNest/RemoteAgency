Remote Agency

Local Exception Handling

Describes the behavior of handling exception occurred locally accessed by remote code.

Local Exception

When an asset is accessed by remote code, a method is called on service wrapper by a remote proxy for example, if an exception is thrown in the asset code block, it may not easy to be caught by user code. By default, Remote Agency suppresses all this kind of exception on the remote site, and sends them back to the caller instead.

If those exception need to be handled on the remote site also, marks the asset or the interface with LocalExceptionHandlingAttribute with LocalExceptionHandlingMode (including the parameter of LocalExceptionHandlingAttribute) other than Suppress.

Throw will make the exception thrown in place. Redirect will let the exception transmit through ExceptionRedirected. If no handler is registered, exception will be thrown.

The default setting is the same as interface when this attribute absents on asset. The default setting is Suppress when this attribute absents on interface.

In particular, the exception thrown while processing system level special messages like disposing of proxy or service wrapper and ping test, will always be redirected through ExceptionRedirected, unless no handler is registered.

Note: This is only for the code that is being accessed by remote code. Exception thrown by the code within the local site, for example the problem occurred in proxy class while accessing a remote method, will not be affected by this setting.

Two-way parameters and properties

When exception is thrown, no return other than exception will be returned by default.

Values of all parameters and properties marked by Two-way attributes with IsIncludedInReturning set to true/True will be transferred and set to local before exception is thrown on the local site.
