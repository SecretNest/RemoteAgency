Remote Agency

Concept

There are several concepts of Remote Agency.

Concept

In the new structure, there are some concepts involved.

  • Site: an instance of Remote Agency which hold objects of proxies and service wrappers and serializing communication from and to another site.

  • Caller: The user code which is wrote for accessing the service object.

  • Proxy: The object created by Remote Agency to work act as the target service object, communicating with the caller code.

  • Service Wrapper: The object created by Remote Agency to work act as a caller, communicating with the service object.

  • Service Object: The code which is designed to be called by user code to complete the corresponding work.

  • Service Contract Interface: An interface which contains the operations which need to be exposed by the service object to the caller.

  • Serialized Messages: Messages representing communication between instances of Remote Agency which are serialized for transferring.

While using Remote Agency, some more concepts are not included above.

  • Asset: Methods, events and properties in a service contract interface.

  • Instance: When not used as "instance of Remote Agency", represents an instance of proxy or service wrapper. Usually, you may see it in the word Instance Id which is the id of the instance of proxy or service wrapper managed by a instance of Remote Agency.

  • Serializer: A serialization solution including a serializer and a deserializer for handling serialized messages and an entity builder for creating class of each entity.
