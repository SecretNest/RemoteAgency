Instance

To use Remote Agency, developer need to create at least one instance of Remote Agency. By accessing the method provided by the instance of Remote Agency, the instance of proxy and service wrapper of the interface provided can be generated.

  • Serializer is required for Remote Agency. When creating the instance of Remote Agency, a serializer needs to be specified.

    • When using JSON serializer, calls CreateWithJsonSerializer to create the instance.

    • When using Binary serializer, calls CreateWithBinarySerializer to create the instance.

       Security Note

      BinaryFormatter is included in Binary Serializer, which is dangerous and should not be used. See Binary Formatter Warning for details.

    • When using other serializer, calls CreateTSerialized, TEntityBase or Create to create the instance with the serializer specified.

    See Serializer for details.

    When siteId is set to Guid.Empty or absent, a randomized value will be generated instead.

  • When CreateProxy or CreateServiceWrapper being called, Remote Agency tries to locate the built assembly in current AppDomain by name and create the instance of the proxy or service wrapper. If absents, a new assembly will be created to hold the classes to be created.

    Each instance of proxy has an id, which is determined when it is created. If the id needs be specified explicitly, calls CreateProxy (CreateProxy) or CreateProxyTInterface. Or, the randomized id will be generated and output when calling CreateProxy (CreateProxy) or CreateProxyTInterface.

    Each instance of service wrapper has an id, which is determined when it is created. If the id needs be specified explicitly, calls CreateServiceWrapper (CreateServiceWrapper) or CreateServiceWrapperTInterface. Or, the randomized id will be generated and returned when calling CreateServiceWrapper (CreateServiceWrapper) or CreateServiceWrapperTInterface.

    See Assembly Building for details.

  • When message filtering is required, see Message Filtering for details.

  • The routing and transmission of messages between instances of Remote Agency are not resolved by Remote Agency. You may want to check Remote Hub or use any other solution.

  • Ping can be used for network detection. When network load balancing is required, it can also be useful to get a particular service wrapper from virtual.

  • When the instance of proxy or service wrapper with events registered is disposed, notification and remedial mechanism to deal with communication failures are described in Dispose notification.

  • When hosting instances of proxy or service wrapper in a mixed environment, some components may be located in the same Remote Agency instance, but some are not, check the Consistency for details.
