Remarks

When initializing with waitForThread set to , a free thread is created for this scheduler.

When initializing with waitForThread set to , SequentialScheduler.Run() should be called from the thread which intends to be used for this scheduler before processing required by any interface.

Caution: The object specified by taskScheduler should be disposed by user code if is returned.

For details please refer to https://github.com/SecretNest/SequentialScheduler/.

This event is not present in Neat release.
