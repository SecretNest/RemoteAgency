Attributes used by Remote Agency

Remote Agency allows users to customize the behavior of Remote Agency by providing attributes.

This topic contains the following sections:

  • Interface Level
  • Asset Level
  • Parameter Level
  • Parameter Level for Event
  • Attribute Pass Through

  • Example

Attributes can be marked on the interface and its related delegate for customizing behavior of Remote Agency.

Interface Level

Name

Description

Remark

ThreadLockAttribute

Designate the task scheduler should be used while accessing assets within.

LocalExceptionHandlingAttribute

Specifies the local exception handling mode.

Can also be used on asset level which has higher priority.

OperatingTimeoutTimeAttribute

Specifies the timeout settings of this asset.

Can also be used on asset level which has higher priority.

Asset Level

Name

Description

Remark

AssetIgnoredAttribute

Specifies the asset is ignored from type building.

AssetOneWayOperatingAttribute

Specifies the operating should be one-way. No response is required.

PropertyGetOneWayOperatingAttribute

PropertyGetOneWayOperatingAttribute

Valid on property only.

ReturnIgnoredAttribute

Specifies the return value of the asset is ignored from type building.

Can also be used on return value level which has higher priority.

Can also be used on delegate level when applying on event which has lower priority.

AsyncMethodAttribute

Specifies this method is asynchronous.

Valid on method only.

Parameter Level

Name

Description

Remark

ParameterIgnoredAttribute

Specifies the parameter should or should not be transferred to remote site. If this attribute absent, the default behavior is transferring all parameters.

Valid on parameter of asset, including method and delegate of event.

ParameterReturnRequiredAttribute

Specifies the parameter should be send back to the caller.

Valid on parameter of asset, including method and delegate of event. The parameter should be marked with "Ref / ByRef" or "out / Out".

ParameterReturnRequiredPropertyAttribute

Specifies a property or field which value should be send back to the caller is contained in the parameter.

Valid on parameter of asset, including method and delegate of event, and property. When specifying this on properties, only set operating will be affected. The parameter should not be marked with "Ref / ByRef" or "out / Out".

ReturnRequiredPropertyHelperAttribute

Specifies the property is a helper for accessing property of a parameter.

Valid on property within helper class.

Parameter Level for Event

Due to no parameter or return value is directly coded while defining an event, some related attribute need to be marked on asset or delegate when required.

Name

Description

Remark

EventParameterIgnoredAttribute

Specifies a parameter of the event should or should not be transferred to remote site.

Valid on event only.

EventParameterReturnRequiredAttribute

Specifies a parameter of the event should be send back to the caller.

Valid on event only.

EventParameterReturnRequiredPropertyAttribute

Specifies a property or field which value should be send back to the caller is contained in a parameter of the event.

Valid on event only.

Attribute Pass Through

Specifies attributes should be marked on and in the created proxy class. It's required when passing the proxy instance to user code which need to check attributes on and in it.

Name

Description

Remark

AttributePassThroughAttribute

Specifies an attribute should be marked at the same place of built proxy class.

Valid on interface, method, event, property, parameter, generic parameter and return value.

AttributePassThroughIndexBasedParameterAttribute

Specifies a parameter value to be passed while constructing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

AttributePassThroughPropertyAttribute

Specifies a property value to be passed while initializing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

AttributePassThroughFieldAttribute

Specifies a field value to be passed while initializing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

Serializer Required Additional Attributes

The chosen serializer may contain some functions that can be customized. When required, serializer defines several attributes and specify the types of them through AssetLevelAttributeBaseType, DelegateLevelAttributeBaseType, InterfaceLevelAttributeBaseType and ParameterLevelAttributeBaseType. When these 4 properties are not set to null / Nothing, attributes with the type specified or its derived classes marked on the object will be collected and transferred to the entity type builder of serializer.

The built in serializers, binary and Json, do not have any additional attributes required.

For details, check attributes for entity types.

Example

C#
VB
Copy
[ThreadLockAttribute(ThreadLockMode.SynchronizationContext)] public interface IMyService
{
    //...
}
<ThreadLockAttribute(ThreadLockMode.SynchronizationContext)> Public Interface IMyService
{
    '...
}
indicates all assets within this interface on server side should be accessed only in synchronization context, instead of the thread passing the message to Remote Agency.
C#
VB
Copy
[CustomizedClassName(assemblyName : "MyApp")] public interface IMyService
{
    //...
}
<CustomizedClassName(, , "MyApp")> Public Interface IMyService
{
    '...
}
replaces the default name of assembly to be built with "MyApp".
