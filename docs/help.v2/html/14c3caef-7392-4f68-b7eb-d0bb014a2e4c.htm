Attributes

Remote Agency allows users to customize the behavior of Remote Agency by providing attributes.

This topic contains the following sections:

  • Interface Level

Attributes can be marked on the interface and its related delegate for customizing behavior of Remote Agency.

Interface Level

Name

Description

Remark

Can also be used on asset level which has higher priority.

Parameter Level

Name

Description

Remark

ParameterIgnoredAttribute

Specifies the parameter should or should not be transferred to remote site. If this attribute absent, the default behavior is transferring all parameters.

Valid on parameter of asset, including method and delegate of event.

ParameterReturnRequiredAttribute

Specifies the parameter should be send back to the caller.

Valid on parameter of asset, including method and delegate of event. The parameter should be marked with "Ref / ByRef" or "out / Out".

ParameterReturnRequiredPropertyAttribute

Specifies a property or field which value should be send back to the caller is contained in the parameter.

Valid on parameter of asset, including method and delegate of event. The parameter should not be marked with "Ref / ByRef" or "out / Out".

ReturnRequiredPropertyHelperAttribute

Specifies the property is a helper for accessing property of a parameter.

Valid on property within helper class.

Parameter Level for Event

Due to no parameter or return value is directly coded while defining an event, some related attribute need to be marked on asset or delegate when required.

Name

Description

Remark

EventParameterIgnoredAttribute

Specifies a parameter of the event should or should not be transferred to remote site.

Valid on event only.

EventParameterReturnRequiredAttribute

Specifies a parameter of the event should be send back to the caller.

Valid on event only.

EventParameterReturnRequiredPropertyAttribute

Specifies a property or field which value should be send back to the caller is contained in a parameter of the event.

Valid on event only.

Attribute Pass Through

Specifies attributes should be marked on and in the created proxy class. It's required when passing the proxy instance to user code which need to check attributes on and in it.

Name

Description

Remark

AttributePassThroughAttribute

Specifies an attribute should be marked at the same place of built proxy class.

Valid on interface, method, event, property, parameter, generic parameter and return value.

AttributePassThroughIndexBasedParameterAttribute

Specifies a parameter value to be passed while constructing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

AttributePassThroughPropertyAttribute

Specifies a property value to be passed while initializing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

AttributePassThroughFieldAttribute

Specifies a field value to be passed while initializing an instance of attribute.

Valid on an object marked with the same AttributeId of AttributePassThroughAttribute.

Customized Name

Names are chosen automatically by default. Specified name is used instead when attributes below are marked.

Name

Description

Remark

CustomizedAssetNameAttribute

Specifies the name of proxy, service wrapper and assembly to be generated.

Valid on interface only.

CustomizedAssetNameAttribute

Specifies the name of the asset.

Valid on method, event and property.

CustomizedMethodEntityNameAttribute

Specifies the names of the entity classes generated for holding parameters and return value of this asset.

Valid on method only.

CustomizedEventEntityNameAttribute

Specifies the names of entity classes generated for event adding, removing and raising.

Valid on event only.

CustomizedEventParameterEntityPropertyNameAttribute

Specifies property name for a parameter in entity class of the event.

Valid on event only.

CustomizedPropertyGetEntityNameAttribute

Specifies the name of the entity class generated for holding the request and response of getting this property.

Valid on property only.

CustomizedPropertyGetResponsePropertyNameAttribute

Specifies property name in entity class generated for holding the response of getting this property.

Valid on property only.

CustomizedPropertySetEntityNameAttribute

Specifies the name of the entity class generated for holding the request and response of setting this property.

Valid on property only.

CustomizedPropertySetRequestPropertyNameAttribute

Specifies property name in entity class generated for holding the request of setting this property.

Valid on property only.

CustomizedParameterEntityPropertyNameAttribute

Specifies property name for the parameter in entity class.

Valid on parameter of method and delegate of event only.

CustomizedReturnValueEntityPropertyNameAttribute

Specifies property name for return value in entity class.

Valid on method, event, delegate of event, and return value of method and delegate.

Serializer Required Additional Attributes

The chosen serializer may contain some functions that can be customized. When required, serializer defines several attributes and specify the types of them through AssetLevelAttributeBaseType, DelegateLevelAttributeBaseType, InterfaceLevelAttributeBaseType and ParameterLevelAttributeBaseType. When these 4 properties are not set as null / Nothing, attributes with the type specified or its derived classes marked on the object will be collected and transferred to the entity code builder of serializer.

The built in serializer, binary and Json, do not have any additional attributes required.
