Disposing Notification

Notification could be generated and sent when proxy or service wrapper object disposed.

Automatic Generating

Since the registration of events needs to be consistent between the Proxy and Service Wrapper, when the Proxy or Service Wrapper is disposed, if any event is registered, a message will be sent to the remote site to notify the disposing happened. The message will be sent automatically while disposing.

Manual Notifying

But when exception is thrown while disposing or error occurred in network transportation for that message, the notification should be sent manually by calling OnRemoteProxyClosing (including OnRemoteProxyClosing) and OnRemoteServiceWrapperClosing (including OnRemoteServiceWrapperClosing).
