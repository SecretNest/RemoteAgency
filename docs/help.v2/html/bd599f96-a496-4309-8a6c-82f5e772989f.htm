Instance

To use Remote Agency, developer need to create at least one instance of Remote Agency. By accessing the method provided by the instance of Remote Agency, the instance of proxy and service wrapper of the interface provided can be generated.

    • Serializer is required for Remote Agency. When creating the instance of Remote Agency, a serializer needs to be specified. See Serializer for details.

    • When CreateProxy or CreateServiceWrapper being called, Remote Agency tries to locate the built assembly in current AppDomain by name and create the instance of the proxy or service wrapper. If absents, a new assembly will be created to hold the classes to be created. See Assembly Building for details.

    • When message filtering is required, see Message Filtering for details.

    • The routing and transmission of messages between instances of Remote Agency are not resolved by Remote Agency. You may want to check Remote Hub or use any other solution.

    • Ping can be used for network detection. When network load balancing is required, it can also be useful to get a particular service wrapper from virtual.

    • When the instance of proxy or service wrapper with events registered is disposed, notification and remedial mechanism to deal with communication failures are described in Dispose notification.

    • When hosting instances of proxy or service wrapper in a mixed environment, some components may be located in the same Remote Agency instance, but some are not, check the Consistency for details.
