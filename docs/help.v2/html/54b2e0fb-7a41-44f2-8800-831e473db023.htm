Welcome to the Remote Agency

Remote Agency is built for making the communicating among components in different computers easier.

Remote Agency can create proxy objects based on one interface file which should be implemented by a remote class, and serializing the accessing between the proxy and the real service object.

NOTE: Network transportation is not included in Remote Agency. The developer should take care about the sending and receiving messages among Remote Agency sites, e.g. using Remote Hub.

Getting Started

Welcome to Remote Agency.

Remote Agency aims to a part of the technical solution for accessing a remote object, only management and serialization, but not network transportation.

Remote Hub, which works in pair with Remote Agency, is ready for taking care of network transportation required by Remote Agency.

The things user should handle:

  • User code as usual;

  • Server object as usual;

  • A service contract interface, including all methods, properties and events, which clients should access;

  • Reference to Remote Agency;

  • The transportation among Remote Agency Manager objects, using Remote Hub for example.

The things Remote Agency cares:

  • Creating types based on the service contract interfaces (access modifier must be public);

  • Creating proxy and service wrapper objects;

  • Handling the serialization and packing process;

  • Catching and playing back exceptions on remote sites; etc.
