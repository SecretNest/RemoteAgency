Attributes for Entity Types

Attributes marked on the interface and its asset can be transferred to the entity type builder of serializer.

Interface Level

When interface level information need to be passed into entity type builder, mark it through an attribute class.

To do that, an attribute class need to be chosen. If none of the existing attributes meet the requirements, define your own. Mark the attribute on the interface.

And, InterfaceLevelAttributeBaseType of the entity type builder need to be set to the type of the attribute. If multiple attribute types are used, set it to a common parent class.

All attributes on the interface with the type as InterfaceLevelAttributeBaseType or its derived classes will be passed to the entity type builder through the second parameter of BuildEntity. InterfaceLevelAttributes will be filled with all qualified instances.

Asset Level

When interface level information need to be passed into entity type builder, mark it through an attribute class.

To do that, an attribute class need to be chosen. If none of the existing attributes meet the requirements, define your own. Mark the attribute on the asset.

And, AssetLevelAttributeBaseType of the entity type builder need to be set to the type of the attribute. If multiple attribute types are used, set it to a common parent class.

All attributes on the asset with the type as AssetLevelAttributeBaseType or its derived classes will be passed to the entity type builder through the second parameter of BuildEntity. AssetLevelAttributes will be filled with all qualified instances.

Event

If the asset is an event, besides on the event itself, the attribute can be defined on the type of event handler (delegate) also. To do that, mark an attribute on the delegate which is used to define the event.

And, DelegateLevelAttributeBaseType of the entity type builder need to be set to the type of the attribute. If multiple attribute types are used, set it to a common parent class.

All attributes on the delegate with the type as DelegateLevelAttributeBaseType or its derived classes will be passed to the entity type builder through the second parameter of BuildEntity. DelegateLevelAttributes will be filled with all qualified instances.

Parameter Level

When parameter level information need to be passed into entity type builder, mark it through an attribute class.

To do that, an attribute class need to be chosen. If none of the existing attributes meet the requirements, define your own. Attribute can be used on parameter, field and property within parameter entity, property within the helper class of the parameter and return value.

And, ParameterLevelAttributeBaseType of the entity type builder need to be set to the type of the attribute. If multiple attribute types are used, set it to a common parent class.

All attributes on the parameter with the type as ParameterLevelAttributeBaseType or its derived classes will be passed to the entity type builder through the element of Properties in the second parameter of BuildEntity. Within each element, Attributes will be filled with all qualified instances on the parameter pointed by the element. Attribute is the instance of the attribute and Position reflects the position of it.

Built-in Serializers

The built in serializers, binary and Json, do not have any additional attributes required. Therefore, no attribute will be collected and transferred to the entity type builder of those serializers.
