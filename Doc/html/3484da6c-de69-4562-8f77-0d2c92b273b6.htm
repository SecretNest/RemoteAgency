Target Site Management

The way to find the site required.

General

By mapping the instance id and site id, when an managed object need to send a message to other object in other site, the site id can be queried by target instance id.

When the site id cannot be found in this mapping table, a default value will be chosen for target site id.

This mechanism is only for the active message, i.e the message send actively, usually the request message. The passive message which responds the request, will be sent back to the sender of the request.

Managing target sites

To get or set the default site id of target: DefaultTargetSiteId

To add a target site mapping: AddTargetSite

To remove a mapping added previously: RemoveTargetSite

To remove all mappings: ResetTargetSite

To query the target site id: GetTargetSiteByInstanceId

To get mappings: GetInstanceIdBySite, GetAllInstancesGroupedBySiteId or GetAllTargetSites
